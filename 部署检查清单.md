# 微信小程序部署检查清单

## 部署前必须检查的项目

### 1. 项目配置
- [ ] 在 `project.config.json` 中填入正确的 AppID
- [ ] 确保 `appid` 字段不为空或占位符
- [ ] 检查 `libVersion` 是否为最新版本

### 2. 文件结构检查
- [ ] 确保所有页面文件都存在：
  - `pages/index/index.js`, `pages/index/index.wxml`, `pages/index/index.wxss`
  - `pages/test/test.js`, `pages/test/test.wxml`, `pages/test/test.wxss`
  - `pages/result/result.js`, `pages/result/result.wxml`, `pages/result/result.wxss`
- [ ] 确保工具文件存在：`utils/wordDatabase.js`
- [ ] 确保根目录文件存在：`app.js`, `app.json`, `app.wxss`, `sitemap.json`

### 3. 代码检查
- [ ] 检查所有 `require()` 路径是否正确
- [ ] 确保没有使用不支持的 API
- [ ] 检查是否有语法错误

### 4. 部署步骤
1. 打开微信开发者工具
2. 导入项目（选择项目根目录）
3. 填入正确的 AppID
4. 点击"编译"检查是否有错误
5. 点击"上传"进行部署

### 5. 常见错误及解决方案

#### 错误：路径不存在
**可能原因：**
- 文件路径大小写不匹配
- 文件实际不存在
- 路径引用错误

**解决方案：**
- 检查所有文件是否真实存在
- 确认路径大小写正确
- 重新检查 `app.json` 中的页面配置

#### 错误：AppID 无效
**解决方案：**
- 在微信公众平台获取正确的 AppID
- 更新 `project.config.json` 中的 `appid` 字段

#### 错误：文件格式错误
**解决方案：**
- 确保 JSON 文件格式正确
- 检查是否有多余的逗号或语法错误

### 6. 调试建议
1. 先在开发者工具中测试编译
2. 查看控制台错误信息
3. 逐步排查问题文件
4. 确保所有依赖文件都存在
